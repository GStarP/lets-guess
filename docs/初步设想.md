# 初步设想

## 背景

我看到一个在社交平台上很火的小游戏，想将其电子化。

游戏内容：找若干图片，提供它们的名称，让不认识它们的人为每张图片选择一个名称，并说明理由。

游戏乐趣：圈内人看圈外人把名称和图片进行各种奇怪的组合，并且给出一些看似有道理的理由，结果最后都是错的。

我希望将其电子化，以实现以下目标：

1. 出题人只要上传图片，并为每张图片标注名称，就可以找人来玩，简化出题人工作。
2. 猜题人选择名称、输入理由即可完成对一张图片的猜测，简化猜题人操作。
3. 系统能够自动生成一张用于分享的长图，用户可以一键保存，省去手动写字、布局、美化的时间，可以快速得到可传播的结果。

## 需求

- 应用应该是一个纯前端应用，只需要把题目数据存储在 localStorage 中，且支持导入导出 JSON 文件
- 应用以移动端优先，兼容 PC 端；使用 tailwindcss 进行样式开发；样式应该扁平化、简洁美观
- 应用分为 出题页、游戏页、结果页 三个页面
  - 出题页
    - 顶部栏：上传按钮，开始游戏按钮，更多按钮
      - 点击更多后弹出菜单：导入、导出、清空
    - 顶部栏下方为已上传的图片，每张图片下方有一个输入框用于填写名称
    - 输入名称不可重复
    - 至少要有 2 张图片，且名称全部填写完毕，才允许点击 开始按钮
    - 点击 开始按钮 后，进入游戏页，从头开始选择（记得要把名称以随机顺序排布，否则玩家只要按顺序选就全对了）
  - 游戏页
    - 顶部栏：返回按钮，已选名称
    - 顶部栏向下纵向排布：当前图片，当前图片下方是理由输入框，可选名称列表
    - 已选名称和理由都预留两行文本的空间，即使没有内容也要留着，防止输入内容后造成布局抖动
    - 点击底部的可选名称可以选择，点击已选择的名称也可以取消；已选择名称 A 后，也可以点击名称 B 替换当前选择
    - 全部选择完毕后，可选名称列表这个区域出现 查看结果 按钮，点击后可进入 结果页
    - 当前图片区域要支持滑动显示上一张/下一张，也要在两侧提供按钮来翻页
    - 点击 返回按钮 后直接回到出题页，不保留游戏中的任何数据
    - 可选名称列表会占满剩余区域，可选名称以 Tag 的形式横向排布（可换行），超出区域高度则纵向滚动
  - 结果页
    - 顶部栏：返回按钮，“正确数：X/XX”，下载按钮
    - 顶部栏下方为系统生成的结果图片，直接用 canvas 渲染图片
    - 点击 返回按钮 后回到出题页
    - 点击下载按钮后，下载结果图片
    - 结果图片，包含所有图片，玩家为每张图片选择的名称和填写的理由，以及选择是否正确（用一种影响较小的方式体现）
      - 结果图片如何设计我还没有想法，请你开动脑筋自己设计一番

## 技术要求

- 始终使用 pnpm 作为包管理器
